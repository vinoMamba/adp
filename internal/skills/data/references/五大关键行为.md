# 五大关键行为

## 总览

五大关键行为用于提升大客户关系、减少项目不确定性，并为采购动机和决策链验证创造场景。

| 行为 | 主要目的 |
|---|---|
| POC | 高频接触客户，验证产品方案，建立管理员和技术负责人关系 |
| 高层拜访 | 提升我方组织影响力，争取客户高层支持 |
| 总部参观 | 把客户带到我方或钉钉主场，增强信任和关系深度 |
| 沙龙峰会 | 借活动场景邀请客户出办公室，培养认知和 Coach |
| 样板点参观 | 通过同行标杆降低客户顾虑，推动需求具体化 |

## POC

适用动作：

- 专属钉 POC。
- TB POC。
- 其他大颗粒新产品 POC。

判断重点：

- 是否拿到真实使用部门。
- 是否接触到主管理员和技术负责人。
- 是否形成高频拜访机会。
- 是否沉淀使用反馈、痛点和扩展场景。

## 高层拜访

适用动作：

- 带我方管理层拜访客户 CXO 或有决策权客户。
- 带钉钉城市经理、SA、大区总拜访客户高层。
- 用对等管理层关系提出总部参观、峰会、项目支持、测试合作等请求。

## 总部参观

适用动作：

- 邀请中层客户到本地办公室或阿里中心交流。
- 邀请客户老板或高层到杭州阿里 / 钉钉总部。
- 配合商务接待、主题交流和案例讲解。

## 沙龙峰会

适用动作：

- 邀请中基层客户参加主题沙龙。
- 邀请 CXO 参加行业高端沙龙。
- 邀请最高层或重要决策人参加总部大型峰会。

## 样板点参观

适用动作：

- 邀请客户参观同行或同城标杆客户。
- 安排业务负责人做同行交流。
- 用真实案例验证方案可行性和价值。

## 记录要求

每个关键行为都要记录：

- 行为名称
- 目标人物
- 目的
- 负责人
- 时间
- 结果
- 对决策链、客情或机会点的影响
