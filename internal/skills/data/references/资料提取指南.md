# 资料提取指南

收到新资料时，按以下步骤将信息提取到知识页。

## 总原则

- 一次只处理一份资料，处理完再处理下一份。
- 每条提取结果必须标注来源文件和日期。
- 高风险事实必须回溯到原始资料或本轮最新工具结果，不能只引用 `客户知识库/*.md`、`输出/*-ADP.md`、`更新日志.md`。
- 预算、成本、报价、合同金额、账号个数、用户数、日期、项目阶段、采购流程、审批链、客户角色和客情积分等字段，必须记录原始来源、资料日期和字段/页码/截图位置。
- 区分三种内容：事实（客户说的、公开可查的）、判断（基于事实推出的）、假设（尚无证据的）。
- 不确定的内容写入对应知识页的"待验证"区域，不要编造。

## Grounding 操作

1. 先用 `客户知识库/来源登记.md` 查找可能来源，再打开对应 `原始资料/` 文件核对。
2. 如果用户本轮提供了新的工具结果、截图、表格、系统导出或合同/报价单，以本轮资料为最新来源。
3. 如果知识页和原始资料数字不一致，优先原始资料；若原始资料之间也冲突，写明冲突并列入待验证。
4. 如果找不到高风险事实的原始来源，写“待回溯原始来源”或“待确认”，不要沿用中间层汇总里的数字。
5. 写入知识页时，`来源` 或 `依据` 字段应写到可回溯粒度，例如“报价单 2026-06-08，第 2 页，账号数字段”。

## 机会质量五维提取口径

只要资料涉及明确 PPL、销售目标、报价或推进中的项目，就同步提取以下五类信息到 `机会与动机.md` 的「机会质量五维评估」。这张表用于内部判断，不直接原样输出到最终 ADP：

| 维度 | 需要提取的信息 | 证据要求 |
|---|---|---|
| 客户的业务及行业分析 | 行业政策、趋势、技术变化、客户业务变化、项目与外部变化的关系 | 公开来源或客户交流记录；写清已交流对象 |
| 使用部门会参与决策 | 业务部门、使用部门、具体使用场景、是否直接接触业务负责人或一线使用者 | 拜访纪要、会议参与人、客户原话或 POC 反馈 |
| 产品解决方案匹配度 | 方案是客户需求/场景驱动，还是我方产品方案驱动；功能满足谁的什么场景和目标 | 方案文档、客户反馈、POC 数据或需求确认 |
| 预计金额及对应决策者 | 预计金额、预算来源、审核层级、审批链、EB/预算/采购/技术角色、价值认可证据 | 报价单、客户口头反馈、审批信息；未确认不得写成事实 |
| 预计时间关联紧迫度 | 预计成交时间、客户推进节奏、紧迫性来源、是否有政策/业务/预算/上线窗口，我方是伴随同行还是跟随者 | CRM 阶段、客户时间要求、下一步约定或项目倒排计划 |

信息不足时写入「待验证动作」，不要留空，也不要用销售主观判断替代客户证据。

## 知识页到最终 ADP 的映射

- `现状.md` 的「现网数字化情况」直接映射到最终 ADP 第三节「数字化现状」的 Markdown 表格。
- `人物与决策链.md` 的关系判断映射到最终 ADP 第二节：保留人员关系图，并在图下补一段总结性描述。
- `人物与决策链.md` 的客情判断映射到最终 ADP 第五节时，只输出简表，不输出客情图。
- `行动计划.md` 的「产品线近期规划」映射到最终 ADP 第七节，最终只保留“计划时间、产品、对接部门”三个字段。
- `现状.md` 的「已购买的钉钉产品」映射到最终 ADP 第八节，不写其他厂商产品。
- 所有 `来源`、`依据`、页码、截图位置、来源链接、来源文件名仅保留在知识页和来源登记中；映射到最终 ADP 时不输出这些溯源字段。

## 按资料类型的提取方法

### 公开调研

来源：年报、官网、新闻、行业报告、招投标公告。

提取目标：

| 信息 | 写入知识页 | 提取方法 |
|---|---|---|
| 主营业务、营收、行业排名、子公司 | 客户画像.md | 摘录关键数据，附来源链接 |
| 数字化转型战略、IT 采购新闻 | 现状.md | 记录公开的系统、厂商、项目名称 |
| 组织变动、高管任命 | 人物与决策链.md | 记录职位变动，不推测影响力 |
| 行业政策、趋势、技术变化、招标公告 | 机会与动机.md | 记录外部影响、机会窗口和对应 PPL，附来源链接 |

### 拜访纪要

来源：客户拜访记录、会议纪要、微信对话总结。

提取目标：

| 信息 | 写入知识页 | 提取方法 |
|---|---|---|
| 客户提到的痛点和需求 | 机会与动机.md | 原话记录 + 归纳动机 |
| 提到的系统、厂商、使用情况 | 现状.md | 按系统逐条记录，标注是客户说的 |
| 出席人、职位、态度 | 人物与决策链.md | 记录接触事实，判断客情等级积分需多次验证 |
| 活动空间、信息质量、立场程度 | 人物与决策链.md | 按客情等级积分表记录：工作/社交/半私人/私人空间，公开/有效/私密/误导信息，反对/中立/支持等立场 |
| 项目进展与客户关系进展 | 人物与决策链.md | 有明确项目时写项目进展+客户关系进展；没有明确项目时写客户关系进展 |
| 客户透露的预算、时间线 | 机会与动机.md | 标注"客户口头反馈"而非事实 |
| 使用部门、业务部门、具体场景 | 机会与动机.md | 记录是否直接接触业务部门、谁提出需求、场景是什么 |
| 下一步行动约定 | 行动计划.md | 记录目标人物、目的、负责人、时间 |

### 方案报价

来源：方案文档、报价单、竞标文件。

提取目标：

| 信息 | 写入知识页 | 提取方法 |
|---|---|---|
| 产品线匹配情况 | 机会与动机.md | 对应 PPL 机会 |
| 客户反馈和修改意见 | 机会与动机.md | 记录购买标准变化 |
| 方案功能与客户场景匹配 | 机会与动机.md | 区分客户需求驱动还是我方方案推动，写清受益人和目标 |
| 报价金额、商务条件、审批链 | 行动计划.md / 机会与动机.md | 记录金额、预算/采购/审批角色和客户价值认可证据 |

### CRM 记录

来源：CRM 系统导出、销售日报、周报。

提取目标：

| 信息 | 写入知识页 | 提取方法 |
|---|---|---|
| 项目阶段变化 | 行动计划.md | 更新阶段状态 |
| 历史对接记录 | 现状.md | 补充时间线 |
| 关键人变动 | 人物与决策链.md | 更新人员信息，并检查决策链人员是否全部进入客情评分表 |
| 成交时间、推进节奏、紧迫性 | 机会与动机.md / 行动计划.md | 记录预计成交时间、客户是否着急、节奏依据和下一步验证 |

## 提取后的必做动作

每次提取完成后：

1. 更新 `来源登记.md`：记录资料名称、类型、日期、权威层级、影响的知识页、关键字段。
2. 更新 `更新日志.md`：记录本次更新的时间、动作和判断变化。
3. 检查 `输出/客户名称-ADP.md` 是否需要同步更新。如果知识库中有实质性变化（新增人物、机会变化、客情判断变化），同步更新 ADP 对应章节。
